# 秒杀优惠券系统 - 支付流程设计文档

## 一、现状分析

### 1.1 当前架构（实时扣款模式）
```
用户下单 → Lua原子扣减库存 → MQ异步创建订单 → 返回成功 → 用户获得优惠券
                                    ↓
                            [立即扣减余额]
```

**特点：**
- 用户下单时立即扣减余额
- 无支付等待环节
- 订单状态：NORMAL(1)、CANCEL(2)
- 无超时自动取消机制

### 1.2 目标架构（支付等待模式）
```
用户下单 → Lua检查库存 → 创建待支付订单 → 进入支付环节 → [等待支付/超时取消]
                                    ↓
                          ┌─────────┴─────────┐
                          ↓                   ↓
                    [用户支付成功]        [超时未支付/取消]
                          ↓                   ↓
                    MQ异步创建订单        回滚Redis库存
                    扣减用户余额          恢复库存
                          ↓                   ↓
                    用户获得优惠券       库存返还
```

## 三、核心流程设计

### 3.1 用户下单流程（改造）

```
┌──────────────────────────────────────────────────────────────┐
│                      用户下单流程                            │
└──────────────────────────────────────────────────────────────┘

1. [用户点击抢购]
         ↓
2. 验证用户资格 (verifyUserLevel)
         ↓
3. 检查秒杀活动时间 (isSeckillActive)
         ↓
4. ========== Lua脚本原子操作 ==========
   - 检查库存 stock > 0
   - 检查一人一单 sismember
   - 预扣减库存 (stock - 1)
   - 写入追踪日志
   ====================================
         ↓
5. [库存不足] → 返回"库存不足"
         ↓
6. [检查余额是否充足] (余额/支付宝/微信预授权)
         ↓
7. [余额不足] → 返回"余额不足"
         ↓
8. 创建【待支付】订单 (status=1)
         ↓
9. 记录订单超时时间 (now + 15分钟)
         ↓
10. 发送【延迟消息】(延迟15分钟)
         ↓
11. 返回订单信息 (orderId, payAmount, payDeadline)
         ↓
12. 前端跳转支付页面
```

### 3.2 支付流程

```
┌──────────────────────────────────────────────────────────────┐
│                      支付流程                                 │
└──────────────────────────────────────────────────────────────┘

【方式一：余额支付（同步）】
┌─────────────────────────────────────┐
│  1. 用户选择余额支付                  │
│  2. 冻结用户余额（而非直接扣减）       │
│  3. 发送【同步支付成功消息】          │
│  4. 进入订单完成流程                  │
└─────────────────────────────────────┘

【方式二：支付宝/微信支付（异步）】
┌─────────────────────────────────────┐
│  1. 用户选择支付宝/微信               │
│  2. 调用第三方支付API，创建支付订单    │
│  3. 返回支付二维码链接                │
│  4. 用户扫码支付                      │
│  5. 第三方回调通知 → 我们的回调接口    │
│  6. 发送【异步支付成功消息】           │
│  7. 进入订单完成流程                  │
└─────────────────────────────────────┘

【订单完成流程（支付成功后）】
┌─────────────────────────────────────┐
│  1. 消费支付成功消息                  │
│  2. 更新订单状态：待支付→已支付       │
│  3. 记录支付信息(pay_type, pay_time) │
│  4. 扣减用户余额（余额支付场景）      │
│  5. 释放库存预扣（如果用了预扣机制）   │
│  6. 发送【创建优惠券消息】→ Kafka     │
│  7. 给用户发放优惠券                  │
└─────────────────────────────────────┘
```

### 3.3 超时取消流程

```
┌──────────────────────────────────────────────────────────────┐
│                    超时取消流程                               │
└──────────────────────────────────────────────────────────────┘

【方案一：延迟队列（推荐）】
┌─────────────────────────────────────────────────────────────┐
│  1. 创建订单时，发送【延迟消息】到延迟队列                    │
│     - 延迟时间 = 15分钟                                      │
│     - 消息内容 = orderId, userId, voucherId                  │
│                                                             │
│  2. 延迟消息到期 → 消费者处理                                 │
│                                                             │
│  3. 检查订单状态：                                           │
│     ├─ [已是已支付/已取消] → 忽略，直接返回                   │
│     └─ [仍是待支付] → 执行超时取消逻辑                        │
│                                                             │
│  4. 超时取消逻辑：                                           │
│     ├─ 更新订单状态：待支付 → 已超时                          │
│     ├─ 回滚Redis库存（Lua脚本）                              │
│     ├─ 释放预冻结的余额（如果用了预冻结）                     │
│     └─ 记录对账日志                                          │
└─────────────────────────────────────────────────────────────┘

【方案二：定时任务（备选）】
┌─────────────────────────────────────────────────────────────┐
│  1. 每分钟执行一次定时任务                                    │
│  2. 查询所有【待支付且已超时的订单】                          │
│  3. 批量执行超时取消逻辑                                      │
│                                                             │
│  缺点：存在最多1分钟延迟，有窗口期                            │
└─────────────────────────────────────────────────────────────┘
```

 * 支付成功消费者
 */
@KafkaListener(topics = {PAY_SUCCESS_TOPIC})
public void onPaySuccess(String value, Acknowledgment acknowledgment) {
    try {
        PaySuccessMessage message = JSON.parseObject(value, PaySuccessMessage.class);

        // 1. 更新订单状态为已支付
        voucherOrderService.updateOrderPaid(message.getOrderId(), message.getPayType(),
                message.getTradeNo(), message.getPayTime());

        // 2. 扣减用户余额（余额支付场景）
        if (message.getPayType() == 1) {
            // 余额已冻结，这里确认扣减
            userAccountService.confirmDeduction(message.getUserId(), message.getOrderId());
        }

        // 3. 发送创建优惠券消息（原有逻辑）
        seckillVoucherProducer.sendCreateVoucherMessage(message);

        acknowledgment.acknowledge();
    } catch (Exception e) {
        log.error("支付成功消息处理失败", e);
        // 不ack，消息会重试
    }
}

/**
 * 订单超时消费者（延迟队列）
 */
@KafkaListener(topics = {ORDER_TIMEOUT_TOPIC})
public void onOrderTimeout(String value, Acknowledgment acknowledgment) {
    try {
        OrderTimeoutMessage message = JSON.parseObject(value, OrderTimeoutMessage.class);

        // 1. 检查订单状态
        VoucherOrder order = voucherOrderService.getById(message.getOrderId());

        // 2. 只有待支付状态才处理
        if (order.getStatus() != 1) {
            acknowledgment.acknowledge();
            return;
        }

        // 3. 执行超时关闭
        voucherOrderService.closeOrder(message.getOrderId());

        acknowledgment.acknowledge();
    } catch (Exception e) {
        log.error("订单超时消息处理失败", e);
    }
}

@Override
public Long createPendingOrder(Long userId, Long voucherId, BigDecimal payAmount) {
    // 1. 生成订单ID（雪花算法）
    Long orderId = snowflakeIdGenerator.nextId();

    // 2. 计算支付截止时间
    LocalDateTime payDeadline = LocalDateTime.now().plusMinutes(15);

    // 3. 创建订单（状态=待支付）
    VoucherOrder order = VoucherOrder.builder()
            .id(orderId)
            .userId(userId)
            .voucherId(voucherId)
            .payAmount(payAmount)
            .status(1)  // 待支付
            .payDeadline(payDeadline)
            .createTime(LocalDateTime.now())
            .build();

    voucherOrderMapper.insert(order);

    // 4. 发送延迟超时消息（15分钟）
    orderTimeoutProducer.sendDelayMessage(orderId, userId, voucherId, 15 * 60 * 1000);

    return orderId;
}

@Override
public void closeOrder(Long orderId) {
    // 1. 查询订单
    VoucherOrder order = voucherOrderMapper.selectById(orderId);
    if (order == null || order.getStatus() != 1) {
        return;
    }

    // 2. 更新状态为已超时
    order.setStatus(4);  // 已超时
    voucherOrderMapper.updateById(order);

    // 3. 回滚Redis库存
    redisVoucherData.rollbackRedisVoucherData(...);

    // 4. 释放预冻结余额（如果有）
    if (order.getPayType() != null) {
        accountService.unfreezeBalance(order.getUserId(), order.getPayAmount());
    }

    // 5. 记录对账日志
    voucherReconcileLogService.saveReconcileLog(...);
}

## 七、余额冻结机制（可选优化）

### 7.1 为什么要冻结而非直接扣减

```
场景：用户余额100元，秒杀商品50元

【直接扣减的问题】
用户下单 → 扣减50余额 → 创建订单 → 用户不支付 → 余额无法恢复（需要额外处理）

【冻结机制】
用户下单 → 冻结50余额（可用余额变为50） → 用户支付成功 → 扣减冻结金额 → 完成
                                              ↓
                                      用户取消/超时 → 解冻50余额 → 恢复100
```

### 7.2 用户账户表改造

```sql
ALTER TABLE tb_user_account ADD COLUMN frozen_balance DECIMAL(10,2) DEFAULT 0 COMMENT '冻结余额';
ALTER TABLE tb_user_account ADD COLUMN total_deduction DECIMAL(10,2) DEFAULT 0 COMMENT '累计扣减';
```

### 7.3 冻结/解冻接口

## 九、数据一致性保证

### 9.1 消息可靠性

```
1. 生产者确认：
   - 发送消息后等待broker确认
   - 未确认则重试发送

2. 消费者确认：
   - 手动提交offset
   - 业务处理成功才ack
   - 失败则重试

3. 消息持久化：
   - broker开启持久化
   - 防止消息丢失
```

### 9.2 支付幂等性

```java
@KafkaListener(topics = {PAY_SUCCESS_TOPIC})
public void onPaySuccess(String value, Acknowledgment acknowledgment) {
    // 1. 解析消息
    PaySuccessMessage message = JSON.parseObject(value, PaySuccessMessage.class);

    // 2. 幂等检查：查看是否已处理
    String key = "pay:callback:" + message.getOrderId();
    Boolean alreadyProcessed = redisCache.setIfAbsent(key, "1", 24, TimeUnit.HOURS);

    if (!alreadyProcessed) {
        acknowledgment.acknowledge();  // 已处理过，直接返回
        return;
    }

    try {
        // 3. 执行业务逻辑
        // ...
    } catch (Exception e) {
        // 4. 失败则删除幂等标记，重试
        redisCache.del(key);
        throw e;
    }
}
```

public Result<Long> seckillVoucher(@RequestBody SeckillVoucherDTO dto) {
    // 1. 验证用户资格
    seckillVoucherService.verifyUserLevel(dto.getUserId());

    // 2. 检查秒杀活动
    seckillVoucherService.isSeckillActive(dto.getVoucherId());

    // 3. Lua原子操作：预扣库存
    Long orderId = luaScriptService.preDeductStock(dto.getUserId(), dto.getVoucherId());
    if (orderId == null) {
        return Result.fail("库存不足");
    }

    // 4. 计算价格（实际项目中从优惠券表查）
    BigDecimal payAmount = seckillVoucherService.getSeckillPrice(dto.getVoucherId());

    // 5. 冻结余额（如果余额足够）
    if (!userAccountService.freezeBalance(dto.getUserId(), payAmount)) {
        // 解冻库存
        luaScriptService.rollbackPreDeduct(orderId);
        return Result.fail("余额不足");
    }

    // 6. 创建待支付订单
    Long pendingOrderId = voucherOrderService.createPendingOrder(
            dto.getUserId(), dto.getVoucherId(), payAmount);

    // 7. 返回订单信息
    return Result.success(pendingOrderId);
}
