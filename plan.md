# 项目新命名方案

## 推荐项目名

中文名：**智邻生活 Agent 平台**

英文名：**LocalMind Agent Platform**

建议后端工程名：**localmind-agent-service**

## 命名定位

“智邻生活 Agent 平台”定位为一个面向本地生活场景的 Java 后端智能体项目。它不是单纯的店铺点评、优惠券或秒杀系统，而是在现有本地生活业务能力之上，引入 Agent 能力，让系统能够理解用户意图、结合用户位置、历史消费、店铺信息、优惠券、营业状态等数据，主动完成推荐、导购、探店规划和出行辅助。

英文名 “LocalMind Agent Platform” 中：

- **Local** 表示本地生活、周边门店、附近服务和线下消费场景。
- **Mind** 表示智能决策、用户偏好理解和多条件推理。
- **Agent Platform** 表示项目后续会承载多个智能体能力，而不是只做单一聊天功能。

## 项目介绍

智邻生活 Agent 平台是基于 Java 后端体系构建的本地生活智能体服务平台。项目以商户、用户、优惠券、订单、达人探店、附近门店、Redis GEO、缓存、高并发秒杀、消息队列和分库分表等已有能力为基础，进一步扩展智能推荐导购 Agent、探店出行规划 Agent 等智能化业务模块。

平台的核心目标是让用户可以用自然语言表达真实消费需求，例如人数、时间、距离、预算、门店类型、评分要求、历史偏好等。Agent 会解析用户需求，结合店铺数据、距离范围、评分、预算、优惠券可用性和用户历史消费记录，给出更贴合场景的推荐结果，并在必要时主动说明推荐理由。

## 首期 Agent：智能推荐导购 Agent

面向用户端首页提供一个“推荐小精灵”入口。用户点击后进入智能推荐导购 Agent 页面，可以通过对话描述自己的消费需求。

示例需求：

> 我和我父母现在计划去找一个 KTV，今天下午 5 点到 8 点左右，店铺距离要在 5 公里内。你给我们找一个评分高的，同时费用在 150 元以下的店铺。

Agent 需要完成的核心能力：

- 从自然语言中提取消费类型、时间段、人数关系、距离范围、预算、评分偏好等条件。
- 查询符合条件的附近门店，并综合评分、距离、价格、优惠券等因素排序。
- 结合用户历史 KTV 消费记录，如果存在符合条件且用户曾经消费过的门店，需要向用户说明并询问是否优先考虑这家店。
- 输出推荐理由，而不是只返回店铺列表。

## 后续 Agent：探店出行规划 Agent

面向用户提供探店出行辅助能力，根据用户当前位置和偏好，自动组合周边美食、休闲、娱乐等探店路线。

计划能力包括：

- 根据用户位置、兴趣偏好、预算和时间范围生成周边探店路线。
- 结合优惠券可用门店，优先规划高性价比方案。
- 提醒门店营业状态、排队热度、优惠券使用截止时间等风险点。
- 在数据库设计支持且扩展成本合理的前提下，支持节假日热门商圈游玩攻略；如果当前数据结构不适合支撑该能力，可在后续版本中裁剪。

## 项目命名理由

选择“智邻生活 Agent 平台 / LocalMind Agent Platform”的原因：

- 能覆盖当前已有的本地生活业务，而不是只绑定 KTV 或导购单点功能。
- “Agent 平台”明确表达后续会扩展多个智能体模块。
- 适合 Java 后端项目命名，工程名 `localmind-agent-service` 简洁、服务化、偏后端。
- 不脱离现有本地生活业务中的商户、优惠券、附近门店、订单和用户行为数据基础。
- 后续无论是智能导购、探店路线规划、优惠券推荐，还是节假日商圈攻略，都可以放在同一套项目定位下演进。
